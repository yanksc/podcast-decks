Install
10 / 35
Install: the basics

It’s easier than you think — but don’t install it on your main computer.

The single biggest mistake newcomers make. The agent operates on your machine: it can move files, send messages, hit your calendar. You don’t want that on the laptop your work, banking, and family photos live on.

# a clean machine — old laptop or Mac Mini, anything you can spare
$ curl -sSL openclaw.ai/install | sh
→ fetching latest...
✓ installed
$ openclaw start --agent finn
✓ Finn online
listening on iMessage, calendar, mail
# onboard like a real assistant: tell it who you are, what it owns, what it doesn’t

Onboarding
11 / 35
Onboarding

Set it up like you’re onboarding a real assistant.

The framing change that made everything work. You wouldn’t hand a new hire your laptop and say “figure it out.” You’d give them a job description, an intro doc, access to specific systems, and feedback for the first month.

  1. Write the job description.

    What does this agent own? What does it explicitly not own? What’s its tone? The one-pager you’d give a new hire.

  2. Give it a name.

    Polly, Finn, Sam — not because it’s cute. Because you’ll remember which one to talk to about which thing.

  3. Give it scoped access.

    One calendar, one inbox, one folder. Not all of them. The principle of least privilege isn’t paranoia — it’s how trust is built.

  4. Give it feedback for the first week.

    Tell it when it gets the tone wrong. When it makes the wrong call. When it should’ve asked. Like any new hire.

Security
12 / 35
Security & privacy

The risks are real. The mitigations are boring and sufficient.

Most concerns Claire hears are valid in theory and fully addressable in practice. The pattern: scope tightly, separate machines, don’t mix domains.

⚠

The risk

Agent has access to a lot. Wrong action on the wrong system can blow up real things — calendars, emails, money.

▣

The principle

Least privilege. Each agent gets the smallest scope that lets it do its job — and nothing else.

✓

The practice

Dedicated machine. Sandboxed accounts. Test on toy data first. Never the main laptop. Never “just for now.”

None of this is glamorous. All of it is what separates the people who are using AI agents from the people who tweet about them.

incident report

Day one: agent given keys to family calendar. Family calendar deleted within hours.

Recovered from backup, but the lesson was bigger than the data: agents are not consumer apps. They are operators with their hands on real systems. Treat them accordingly from minute one.

Lessons learned the hard way

The first install will break something. Plan for it.

Backups before, sandboxes during, scoped access throughout. The expectation that the first run will go cleanly is what makes it go badly.

Claire’s framing: this is closer to setting up a home server than installing an app. Treat it like sysadmin work, because it is.

Tactic
18 / 35
Tactic

You can run several agents on a single machine.

You don’t need one box per agent. A modest Mac Mini comfortably hosts three or four well-scoped agents. The split is more about domain isolation than CPU.

  1. Group by domain, not by load.

    Family agents on one box. Work agents on another. Specialists on a third. Why? Blast radius if something goes wrong.

  2. Give each its own user / scope.

    Separate macOS user accounts where it matters. Separate API keys, separate calendars, separate inboxes.

  3. Don’t cross the streams.

    Sam (sales) shouldn’t see family iMessage. Finn (family) shouldn’t see customer email. Easier on day one than retroactively.

Three problems you’ll hit. All have workarounds.

Browser limitations

Built-in browsing is rough. Workaround: pair OpenClaw with Exa or Perplexity for real web search.

▢

Memory overload

Context fills up. Agent gets foggy. Workaround: prune memory deliberately. Reset weekly. Smaller context = sharper agent.

▢

Headless management

Mac Minis run without a screen. Workaround: screen-share into them from your laptop. Not glamorous; works.

Workaround
20 / 35
Workaround · web search

For real web access: plug in a specialist.

OpenClaw’s built-in browser isn’t up to heavy research. Claire wires in Exa or Perplexity for the actual searching, then lets the agent reason over the results. The agent is the brain; the search engine is the eyes.

# typical research pattern
user » research X for tomorrow’s call
→ agent dispatches search to Exa
→ Exa returns 8 high-signal sources
→ agent synthesizes, summarizes, links back
✓ briefing in your inbox — 3 minutes total

The pattern generalizes: when a built-in tool is weak, pair the agent with a specialist. The agent’s job is judgment, not every primitive.

Memory
21 / 35
You really have to pull the thread on these tools and you have to spend enough time with them to see not where they are today, but where they are in a week and where they are in a month. — Claire Vo
Memory hygiene

Managing memory is 50% of the job.

Agents that “get worse” over time mostly haven’t. Their memory got bloated. Once you treat memory hygiene as a recurring task — like cleaning a kitchen — the quality stops decaying.

Practical rule: weekly memory review. What does the agent “remember” that’s no longer true? What facts have rotated? What patterns has it picked up that you don’t want?

The meta-skill

The most useful skill for AI agents is management.

Not prompt engineering. Not prompt-engineering school. The actual skill is the same one that makes someone good at running a small team: writing the JD, scoping the role, giving feedback, knowing when to fire and re-cast.

Action List
33 / 35
Action list · what to do Monday

Six steps to your first useful agent.

  1. Find a spare machine.

    Old laptop, Mac Mini, anything. Not your daily driver. This is non-negotiable.

  2. Pick one job, write the JD.

    One page. Title, scope, tone, what it owns, what it doesn’t. Specific.

  3. Install. Onboard. Name.

    Treat it like a new hire’s first day. Slow, deliberate, written down.

  4. Connect to one channel.

    iMessage. One inbox. One calendar. Resist scope creep for the first month.

  5. Live with it for two weeks.

    Note what it does well. Note what breaks. Tweak the JD weekly.

  6. Then — and only then — add a second agent.

    Different job. Different name. Same discipline. The household builds slowly on purpose.
